## What happened

- Cold & dark verified by telemetry (BUS E 0.036 V — transducer noise floor,
  honestly nonzero).
- Deliberate mistake per the vignette: `eps bus.a.tie close --confirm` onto
  dead BUS A → `TRIPPED (inrush)`, FDR `inrush-trip {peak_a: 1332.5,
  limit_a: 150.0}` — within 2% of the SOM's promised ">1000 A, roughly seven
  times rating".
- `eps bus.a.precharge start` → `REFUSED: interlock tie.a must be OPEN` with
  manual citation. **First run stranded here**: no manual section taught the
  `reset` verb (see friction #1). After the fix: RESET → verify OPEN →
  precharge → `precharge-complete` at 2.4 s (τ = 0.6 s, consistent) → tie
  CLOSE clean → all three buses up.
- End state matched SOM 24-30-03 expected indications: BUS A 26.198 V
  (hand-computed 26.21), BAT 1 6.910 A (computed 6.97), BAT 2 5.031 A
  (computed 5.01). MASTER CAUTION clear.
- Breaker pull mid-flight (`eps cb.e1 open`): BAT 1 current 6.9 → 0.03 A
  (cabin was on BUS A via tie — wait, avionics only was on E; current drop
  reflects avionics + tie path shed), BUS E rose to open-circuit 26.76 V.
  Honest symptoms, no God-view leakage.
- Battery contactor open under load: BUS E held charge (loads were shed
  first) — no UNDERVOLT, correctly, because the monitors saw no undervolt.
- Same power-up sequence exercised in the TUI (SYS/EPS station + DOCS
  split): annunciator row, feeder-tree live states, and command bar echoes
  all consistent with teletype output.
- Conformance suite proves the same procedures headlessly (som-24-30-01..04,
  including across-seeds noise robustness).

## Surprise list (emergent-behavior ledger)

- The noise floor makes "verify < 2 V" read 0.036 V instead of 0.000 — an
  unauthored touch that teaches transducer humility on step 1.
- Post-shutdown BUS A read −0.022 V (noise below zero on a dead bus):
  harmless, slightly delightful, exactly what a real transducer does.
- A tripped tie holds the precharge interlock (state machine fallout, never
  explicitly designed): initially surprising, then obviously correct — and
  now documented as a feature in 24-00-00 §3.
